
# Java file systen

___Java gắn chặt cấu trúc code vào cấu trúc thư mục (File System).___

Trong Java, cấu trúc thư mục không chỉ để cho đẹp mà nó là địa chỉ định danh (Identifier) của code.

- Package = Folder: Nếu khai báo package com.example.utils; ở đầu file, thì file đó bắt buộc phải nằm trong thư mục com/example/utils/. Nếu di chuyển file đi chỗ khác mà không đổi dòng code package, chương trình sẽ lỗi ngay.

- Tên File = Tên Class Public: public class User phải nằm trong User.java.

- Import dựa trên Package: Khi import, Java nhìn vào CLASSPATH để dò tìm theo sơ đồ thư mục.

Tên class nên đặt theo PascalCase (quy chuẩn chính thức - Java Language Specification).

Trong một file Java, chỉ có tối đa một class được khai báo là `public`. Nếu có 2 class public, trình biên dịch sẽ báo lỗi.

Nếu một file có một class public, thì tên file phải trùng khớp hoàn toàn với tên của class đó (bao gồm cả việc phân biệt chữ hoa chữ thường).

  > _Java yêu cầu đặt tên public class trùng với tên file để __Tối ưu hóa việc tìm kiếm của trình biên dịch__._
  >
  > _Khi trình biên dịch hoặc JVM cần tìm class Com.Example.Utils, nó chỉ cần tìm đúng file Utils.java thay vì phải quét qua toàn bộ nội dung của mọi file trong thư mục._

Không được dùng private hay protected cho class ngoài.

- Không dùng private: Vì nếu một class là private, không ai (kể cả trình biên dịch) có thể truy cập để khởi tạo nó, dẫn đến việc class đó trở nên vô dụng.

- Không dùng protected: Vì mục đích của protected là để cho các class con ở package khác kế thừa, nhưng ở cấp độ file, Java ưu tiên dùng public hoặc mặc định để quản lý cấu trúc hệ thống.
