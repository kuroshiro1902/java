# var

Từ phiên bản **Java 10**, từ khóa `var` được giới thiệu để giúp code ngắn gọn hơn bằng cách cho phép trình biên dịch tự suy luận kiểu dữ liệu (Type Inference).

---

### 1. Cách hoạt động
Thay vì phải khai báo cụ thể kiểu dữ liệu như `String`, `ArrayList`, hay `int`, bạn chỉ cần dùng `var`. Java sẽ nhìn vào giá trị vế phải để quyết định kiểu của biến.

* **Truyền thống:** `String message = "Hello Java";`
* **Với `var`:** `var message = "Hello Java";` (Java tự hiểu `message` là `String`).

### 2. Các quy tắc quan trọng (Bắt buộc)
* **Phải khởi tạo ngay:** Bạn không thể viết `var x;` rồi mới gán giá trị sau. Trình biên dịch cần giá trị để biết `x` là cái gì.
* **Chỉ dùng cho biến cục bộ (Local Variables):** Chỉ được dùng bên trong các phương thức (hàm), vòng lặp hoặc khối mã.
* **Không dùng làm:** Thuộc tính của Class (fields), tham số hàm, hoặc kiểu trả về của hàm.
* **Kiểu dữ liệu là cố định:** Sau khi đã suy luận xong (ví dụ là `int`), bạn không thể gán một giá trị kiểu khác (như `String`) cho biến đó.



### 3. Ưu và nhược điểm
* **Ưu điểm:** Giảm bớt sự rườm rà khi làm việc với các kiểu dữ liệu phức tạp như `Map<String, List<Integer>>`.
* **Nhược điểm:** Nếu lạm dụng hoặc đặt tên biến không rõ ràng, người đọc code sẽ khó biết biến đó thực chất thuộc kiểu dữ liệu nào.
